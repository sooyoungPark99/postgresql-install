# Oracle 19c 소프트웨어 설치

### 1. OS 버전 확인

Oracle 설치 전 OS 버전과 커널 버전을 확인한다.

```bash
rpm -qa *-release
uname -a
```

| 리눅스 버전 | 권장 Oracle DB |
|-----------|--------------|
| OL 9 / RHEL 9 | 19c (19.19+), 23c |
| OL 8 / RHEL 8 | 19c, 21c |
| OL 7 / RHEL 7 | 12cR2, 18c, 19c |
| OL 6 / RHEL 6 | 11gR2, 12cR1 |

---

### 2. 네트워크 및 호스트 설정

/etc/hosts에 서버 IP와 호스트명을 등록한다.

```bash
vi /etc/hosts
```

```
127.0.0.1   localhost localhost.localdomain
192.168.13.181 ora19c
```

호스트명을 설정한다.

```bash
hostnamectl set-hostname ora19c
hostnamectl status
```

---

### 3. oracle-database-preinstall-19c 패키지 설치

preinstall 패키지를 설치하면 Oracle 설치에 필요한 사전 준비가 자동으로 완료된다.

```bash
yum -y install oracle-database-preinstall-19c
```

preinstall 패키지가 자동으로 처리하는 항목은 아래와 같다.

| 항목 | 내용 |
|------|------|
| 필수 패키지 설치 | glibc, gcc, libaio 등 |
| 커널 파라미터 설정 | /etc/sysctl.conf 자동 구성 |
| oracle 유저/그룹 생성 | oinstall, dba 등 |
| limits 설정 | /etc/security/limits.d/ 자동 구성 |

---

### 4. oracle 유저 그룹 확인 및 설정

그룹 생성 여부를 확인한다.

```bash
cat /etc/group | grep -E 'oinstall|dba|oper'
```

oracle 유저의 기본 그룹과 보조 그룹을 설정한다.

```bash
# 기본 그룹 설정
usermod -g oinstall oracle

# 보조 그룹 추가
usermod -a -G dba,oper,backupdba,dgdba,kmdba,racdba oracle

# 확인
id oracle
```

---

### 5. 커널 파라미터 및 limits 확인

커널 파라미터 적용 여부를 확인한다.

```bash
sysctl -p
```

limits 설정을 확인한다.

```bash
grep -v ^# /etc/security/limits.d/oracle-database-preinstall-19c.conf
```

### 6. 표준 디렉터리 생성 및 권한 설정

```bash
mkdir -p /u01/app/oracle/product/19.3.0/dbhome_1
mkdir -p /u01/app/oraInventory
chown -R oracle:oinstall /u01
chmod -R 775 /u01
```

디렉터리 구조는 아래와 같다.

```
/u01
 └─ app
    ├─ oracle
    │  └─ product
    │     └─ 19.3.0
    │        └─ dbhome_1   (ORACLE_HOME)
    └─ oraInventory
```

| 경로 | 설명 |
|------|------|
| ORACLE_BASE | /u01/app/oracle |
| ORACLE_HOME | /u01/app/oracle/product/19.3.0/dbhome_1 |
| oraInventory | 설치/패치/제거 메타데이터 저장 경로 |

---

### 7. oracle 계정 환경변수 설정

```bash
su - oracle
vi .bash_profile
```

아래 내용을 추가한다.

```bash
export ORACLE_BASE=/u01/app/oracle
export ORACLE_HOME=/u01/app/oracle/product/19.3.0/dbhome_1
export ORA_INVENTORY=/u01/app/oraInventory
export ORACLE_SID=ORA19
export PATH=$PATH:$HOME/bin:$ORACLE_HOME/bin
export LD_LIBRARY_PATH=$ORACLE_HOME/lib:/lib:/usr/lib
export CLASSPATH=$ORACLE_HOME/JRE:$ORACLE_HOME/jlib:$ORACLE_HOME/rdbms/jlib
```

환경변수를 적용하고 확인한다.

```bash
source .bash_profile
env | grep ORA
```

### 8. X11 Forwarding 설정 (GUI 설치 준비)

MobaXterm으로 oracle 유저 접속 후 xclock으로 GUI 동작을 확인한다.

```bash
echo $DISPLAY
xclock
```

xclock 명령어가 없으면 아래와 같이 설치한다.

```bash
yum install -y xorg-x11-apps
```

> X11 Forwarding은 원격 서버의 GUI 화면을 로컬에서 표시하는 기능이다.
> Oracle GUI 설치 프로그램(runInstaller) 실행 시 필요하다.

### 9. 설치 파일 업로드 및 압축 해제

ORACLE_HOME 경로에서 설치 파일의 압축을 해제한다.

```bash
cd /u01/app/oracle/product/19.3.0/dbhome_1
unzip -q ~/LINUX.X64_193000_db_home.zip
```

> 설치 파일(LINUX.X64_193000_db_home.zip)은 Oracle 공식 사이트에서 다운로드한다.
> https://www.oracle.com/database/technologies/oracle-database-software-downloads.html

### 10. Oracle Software 설치 (GUI)

runInstaller를 실행하여 설치를 시작한다.

```bash
cd /u01/app/oracle/product/19.3.0/dbhome_1
./runInstaller
```

설치 옵션은 아래 순서대로 진행한다.

| 단계 | 설정값 |
|------|--------|
| 설치 유형 | Set Up Software Only |
| DB 유형 | Single instance database installation |
| 에디션 | Enterprise Edition |
| Oracle Base | /u01/app/oracle |
| Inventory 경로 | /u01/app/oraInventory (그룹: oinstall) |
| root 스크립트 | 자동 실행 또는 수동 실행 |

### 11. Swap 부족 경고 해결

설치 중 Prerequisite Checks 단계에서 Swap 경고가 발생하면 아래와 같이 Swap을 추가한다.

Swap 상태를 먼저 확인한다.

```bash
free -h
swapon --show
```

Swap 파일을 추가한다 (예: 3GB).

```bash
# Swap 파일 생성
dd if=/dev/zero of=/swapfile bs=1G count=3

# 권한 설정
chmod 600 /swapfile

# Swap 포맷
mkswap /swapfile

# Swap 활성화
swapon /swapfile

# 재부팅 후에도 유지되도록 설정
echo '/swapfile swap swap defaults 0 0' >> /etc/fstab
```

적용 후 확인한다.

```bash
free -h
```

이후 설치를 다시 진행한다.
