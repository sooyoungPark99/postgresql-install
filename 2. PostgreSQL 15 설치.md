# PostgreSQL 15 설치

### 1. 저장소 추가 및 패키지 설치

PostgreSQL 공식 YUM 저장소(PGDG)를 추가하고 패키지를 설치한다.

```bash
# PostgreSQL 저장소 추가
yum install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-7-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# PostgreSQL 15 설치
yum install -y postgresql15-server postgresql15-contrib

# 설치 확인
ls -l /usr/pgsql-15/bin/
```

| 패키지 | 설명 |
|--------|------|
| postgresql15-server | PostgreSQL 서버 패키지 |
| postgresql15-contrib | 확장 및 유틸리티 패키지 모음 |

### 2. 설치 오류 발생 시 저장소 정리

<img width="1032" height="881" alt="image" src="https://github.com/user-attachments/assets/f56644b9-c9c9-4f2a-8bb6-25c7f26f3e17" />

PGDG 저장소가 여러 버전으로 섞이면 404 또는 의존성 오류가 발생할 수 있다.
아래 순서대로 정리 후 재설치한다.

```bash
# 1. pgdg 전체 저장소 비활성화
yum-config-manager --disable pgdg\*

# 2. pgdg15와 공통 저장소만 활성화
yum-config-manager --enable pgdg15 pgdg-common

# 3. 캐시 초기화
yum clean all

# 4. 재설치
yum install -y postgresql15-server postgresql15-contrib
```

### 3. 초기화 및 서비스 기동

```bash
# 데이터베이스 초기화
/usr/pgsql-15/bin/postgresql-15-setup initdb

# 자동 시작 등록
systemctl enable postgresql-15

# 서비스 시작
systemctl start postgresql-15

# 상태 확인
systemctl status postgresql-15
```

> initdb는 데이터 디렉터리를 초기화하는 명령어로 최초 1회만 실행한다.
> 기본 데이터 디렉터리: /var/lib/pgsql/15/data/

### 4. 기본 설정 (비밀번호/외부접속/방화벽)

postgres 사용자 비밀번호를 설정한다.

```bash
su - postgres -c "psql -c \"ALTER USER postgres WITH PASSWORD 'oracle';\""
```

외부 접속을 허용하도록 postgresql.conf를 수정한다.

```bash
vi /var/lib/pgsql/15/data/postgresql.conf
```

```
listen_addresses = '*'
port = 5432
```

클라이언트 인증을 허용하도록 pg_hba.conf를 수정한다.

```bash
vi /var/lib/pgsql/15/data/pg_hba.conf
```

맨 아래에 아래 내용을 추가한다.

```
host    all    all    0.0.0.0/0    md5
```

| 설정 파일 | 역할 |
|---------|------|
| postgresql.conf | 서버 파라미터 설정 (리스닝 주소/포트 등) |
| pg_hba.conf | 클라이언트 인증 및 접속 허용 규칙 |

설정 변경 후 서비스를 재시작한다.

```bash
systemctl restart postgresql-15
```

방화벽에서 5432 포트를 오픈한다.

```bash
firewall-cmd --permanent --add-port=5432/tcp
firewall-cmd --reload
```

### 7. 실습 쿼리

직업별 토탈 월급을 조회한다.

```sql
SELECT job, SUM(sal)
FROM emp
GROUP BY job;
```

EMP/DEPT를 조인하여 이름과 부서 위치를 조회한다.

```sql
SELECT e.ename, d.loc
FROM emp e
JOIN dept d ON e.deptno = d.deptno;
```

JONES보다 월급이 많은 사원을 조회한다.

```sql
SELECT ename, sal
FROM emp
WHERE sal > (
    SELECT sal
    FROM emp
    WHERE ename = 'JONES'
);
```

sal 컬럼에 인덱스를 생성하고 실행계획을 확인한다.

```sql
-- 인덱스 생성
CREATE INDEX emp_sal ON emp(sal);

-- 인덱스 목록 확인
SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'emp';

-- 실행계획 확인
EXPLAIN
SELECT ename, sal
FROM emp
WHERE sal = 3000;
```

> 데이터가 적으면 옵티마이저가 Index Scan 대신 Seq Scan을 선택할 수 있다.
> 테스트 목적으로 Seq Scan을 비활성화하여 확인할 수 있다.

```sql
-- Seq Scan 비활성화 (테스트 목적)
SET enable_seqscan = OFF;

EXPLAIN
SELECT ename, sal
FROM emp
WHERE sal = 3000;

-- 원래대로 복원
SET enable_seqscan = ON;
```
