### 3. VirtualBox 네트워크 브리지 설정

가상머신이 호스트 PC와 동일한 네트워크 대역을 사용하도록 설정한다.

```
VM 선택 → 설정 → 네트워크 → 어댑터 1
다음에 연결됨: 어댑터에 브리지(Bridged Adapter)
```

<img width="1552" height="821" alt="image" src="https://github.com/user-attachments/assets/4676224b-74b5-47a4-9998-859692992845" />

> 브리지 모드로 설정해야 호스트 PC와 동일한 네트워크 대역의 IP를 사용할 수 있다.

### 4. Linux VM 고정 IP 설정

VM 내부에서 고정 IP를 설정한다.

| 항목 | 값 |
|------|-----|
| IP 주소 | 192.168.13.181 |
| 넷마스크 | 255.255.255.0 |
| 게이트웨이 | 192.168.13.1 |
| DNS | 8.8.8.8 |

<img width="807" height="610" alt="image" src="https://github.com/user-attachments/assets/a904154a-cf48-4fd7-8746-7a5ecacc1be2" />

> IP는 윈도우 IP + 50번으로 설정하는 것을 권장한다.

네트워크 서비스를 재시작한다.

```bash
systemctl restart NetworkManager
```

<img width="815" height="696" alt="image" src="https://github.com/user-attachments/assets/f2dd95f2-b100-4f9f-b470-3324f4beaf61" />

### 5. 네트워크 설정 확인

IP가 정상 적용됐는지 확인한다.

```bash
# IP 확인
ip a

# 게이트웨이 확인
ip route

# 외부 통신 확인
ping -c 3 8.8.8.8
```

### 6. PuTTY / MobaXterm 접속

호스트 PC에서 아래 정보로 SSH 접속한다.

| 항목 | 값 |
|------|-----|
| Host | 192.168.13.181 |
| Port | 22 |
| Connection type | SSH |

<img width="890" height="579" alt="image" src="https://github.com/user-attachments/assets/b1782ba3-4e3e-486f-9f72-2a31824c0d93" />

접속이 안 되는 경우 아래 항목을 순서대로 확인한다.

1. VM IP 설정 확인 (`ip a`)
2. 브리지 어댑터 설정 여부 확인
3. 방화벽 및 sshd 상태 확인

```bash
systemctl status sshd
firewall-cmd --list-all
```

<img width="831" height="315" alt="image" src="https://github.com/user-attachments/assets/60e2e3bd-bdfe-4692-a8b0-91fac6464d5b" />
