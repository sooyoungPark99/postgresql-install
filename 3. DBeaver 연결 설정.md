# DBeaver 연결 설정

### 1. 원격 접속을 위한 서버 설정 개요

DBeaver에서 PostgreSQL에 접속하려면 아래 3가지 설정이 모두 완료되어야 한다.

| 설정 | 역할 |
|------|------|
| postgresql.conf | PostgreSQL이 외부 IP 요청을 수신하도록 설정 |
| pg_hba.conf | 특정 IP 대역의 접속을 인증 허용 |
| 방화벽(firewalld) | OS 방화벽에서 5432 포트 오픈 |

> 위 3가지 중 하나라도 빠지면 DBeaver에서 접속이 거부된다.

---

### 2. postgresql.conf 수정 (외부 수신 허용)

```bash
vi /var/lib/pgsql/15/data/postgresql.conf
```

아래 내용을 확인 및 수정한다.

```
listen_addresses = '*'
port = 5432
```

> listen_addresses = '*' 로 설정하면 모든 IP에서 접속 요청을 수신한다.

---

### 3. pg_hba.conf 수정 (클라이언트 인증 허용)

```bash
vi /var/lib/pgsql/15/data/pg_hba.conf
```

맨 아래에 아래 내용을 추가한다.

```
host    all    all    192.168.23.0/24    md5
```

> 접속을 허용할 클라이언트 IP 대역은 실습 환경에 맞게 조정한다.
> md5 대신 scram-sha-256 인증 방식도 사용 가능하다.

---

### 4. PostgreSQL 재시작

설정 파일 변경 후 서비스를 재시작한다.

```bash
systemctl restart postgresql-15
```

---

### 5. 방화벽 5432 포트 오픈

```bash
firewall-cmd --permanent --add-port=5432/tcp
firewall-cmd --reload
```

---

### 6. 설정 파일 위치 확인 (참고)

```bash
ps -ef | grep postgres
```

---

### 7. DBeaver 접속

DBeaver를 실행 후 아래 정보로 새 연결을 생성한다.

| 항목 | 값 |
|------|-----|
| Host | 192.168.23.67 |
| Port | 5432 |
| Database | postgres |
| Username | postgres |
| Password | oracle |

연결 후 테이블이 정상적으로 보이는지 확인한다.

> 연결 테스트(Test Connection) 버튼으로 접속 가능 여부를 먼저 확인하는 것을 권장한다.
