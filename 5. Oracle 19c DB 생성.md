# Oracle 19c DB 생성

### 1. 리스너 설정 (listener.ora)

DB 생성 전에 리스너 설정 파일을 준비하고 정상 동작 여부를 확인한다.

```bash
cd $ORACLE_HOME/network/admin
vi listener.ora
```

```
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.13.181)(PORT = 1521))
      (ADDRESS = (PROTOCOL = IPC)(KEY = EXTPROC1521))
    )
  )
```

리스너를 기동하고 상태를 확인한다.

```bash
lsnrctl start
lsnrctl status
```

---

### 2. DBCA로 DB 생성

DBCA를 실행한다.

```bash
dbca
```

설치 옵션은 아래 순서대로 진행한다.

| 단계 | 설정값 |
|------|--------|
| 작업 선택 | Create a database |
| 생성 모드 | Advanced configuration |
| 템플릿 | General Purpose or Transaction Processing |
| DB 이름 | ORA19 |
| SID | ORA19 |
| 컨테이너 DB | 해제 |
| 스토리지 | File System |
| FRA | 활성화 |
| 리스너 | 기존 리스너 선택 |
| Data Vault | 해제 |
| 메모리 | AMM 678MB |
| 샘플 스키마 | 추가 선택 |
| EM Express | 활성화 (포트 5500) |
| 비밀번호 | oracle |
| 생성 옵션 | Create database |

> 비밀번호는 실습 환경이므로 oracle로 설정하지만
> 운영 환경에서는 Oracle 권장 복잡도를 준수한다.

---

### 3. 생성 후 접속 확인

DB 생성이 완료되면 MobaXterm을 새로 열고 oracle 유저로 접속하여 DB 상태를 확인한다.

```sql
sqlplus / as sysdba

select instance_name, status from v$instance;
select name from v$database;
```

| 뷰 | 설명 |
|----|------|
| v$instance | 인스턴스 상태 확인 (OPEN/MOUNT/IDLE) |
| v$database | 데이터베이스 이름 확인 |

---

### 4. 재부팅 후 자동 기동 설정

재부팅 후 DB가 자동으로 올라오지 않을 경우 아래와 같이 수동으로 기동한다.

```sql
startup
```

.bash_profile에서 ORACLE_SID를 실제 SID로 설정하고 편의 alias를 추가한다.

```bash
vi .bash_profile
```

```bash
export ORACLE_SID=ORA19

# 편의 alias
alias sys='sqlplus / as sysdba'
alias alert='cd /u01/app/oracle/diag/rdbms/ora19/ORA19/trace'
```

```bash
source .bash_profile
```

> alert.log는 인스턴스 기동 및 오류 로그를 확인하는 파일이다.
> 문제 발생 시 가장 먼저 확인한다.
